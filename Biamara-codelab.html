Learn how to build a website

Have you ever wondered how all these big company's websites like Google.com, Facebook.com, Youtube.com, Amazon.com, and the rest of them were built? They obviously did not fall from the sky, someone sat down and built them with the same computer that you and I are using everyday. And these people happen to be among the world's richest billionairs today. You too can be among them, it's just a determination. In this tutorial, you are going to learn how to build a website from scratch, it does not require you to have previous experience in programming, but if you can read this to the end carefully, I assure you that you will then gain at least a basic knowledge in web development. However, it is not going to be as easy as reading A B C D, but it is 100 percent possible to become a perfect developer within six months, it only requires your attention and interest by making coding a hobby, that's all.

A functioning website is constructed with multiple machine or code languages, but we will be using the three most used languages, HTML CSS and JavaScript which you must learn in order to have their basic knowledge as a beiginer before we move on to other laguages.

HTML This stands for "Hypertext Markup Language", this is used to build the srtucturer of web pages. It has elements, attributes, tags, entities, etc. Think of HTML as the carpenter and the brick-layer who build a house from the foundation to the roofing level without any special design, no one can move in to that house like that.

CSS This stands for "Cascading Style Sheet", you are now going to think of CSS as the tiler and the painter who design and beautify the house. However, the house is not lively yet but one can still move in, in that current state. Which simply means, after the work of CSS in the website, it could be published online and be visited by users, just for reading blog posts.

JavaScript Finally, you can now think of JavaScript as the Plumber and Electrician who make things happen in the house. A plumber connects every pipe to make sure that there is running water, the electrician does his own job to make things function. JavaScript does the same thing in a website, it is a programming language that makes a website interactive.

Now that you have understood the role that HTML, CSS, and JavaScript play in web development, we are now going start, but before we begin, there are three tools that we will be working with. and .

The code editor is where you will write your codes which will be installed on the computer, then, we use the browser to taste our work. There are numerous code editors out there, but I would suggest you go for Visual Studio Code (VS Code), for it is user friendly in programming, especially for beginners. Here is the link to download one, VS Code Editor. You can also download Google Chrome Browser here, Google Chrome. These are the tools that I would recommend you to use as a new programmer.

Alright, now that we have all the required tools at our disposal, we can now start building our new website. Let's take these few steps first.

  • Create a folder on your computer, you can name it anything
  • Inside the folder you've created, create a file with a HTML extension, example: myfirstwebsite.html
  • Create another file with a CSS extension, example: myfirstwebsite.css
  • Lastly, create another file with a JavaScript extension, example: myfirstwebsite.js

Feel free to use any word or name you like, what really matters is the extension, that is how the computer will understand which language it is before processing it.

  • Open the code editor that you have downloaded, click on the icon explorer,
    • from there you can open the folder you created, inside the folder you will find all your files. open the HTML, that is where we are going to start from.

NOTE: You must structure your HTML code like the one shown here with these default codes before you proceed. The default codes will show up immediately when you type ! and press enter. link your CSS file inside the <head> tag the way it shows on my editor, mine is Code-lesson.css, you can see it there.

When you're building a website, the first thing you should build is the . So, let us build a navigation bar, using HTML and CSS.

     
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <body>
  
      <!--Write your HTML codes here, within the 'body' tag--> 

    </body>

We are now creating the navigation bar with four options, each option means a different web page. The rule of HTML is to always use an open and a close tag like this: <opentag> </closetag>. There are numerous HTML tags, but later in this lesson you will learn some of them and their uses.

You have noticed that we used a <nav> tag, that is the container in which the <h1> tag is written, we use 'h1' to write titles on websites. Inside the <nav> tag we also have a <ul> tag which stands for Unordered Lists, in which we wrote the <li> tag, that is the lists of the menu options. The <a> is the anchor or hyperlink tag that links to anonther page or website, it has the attribute href=" ", inside which it has two double quotes where website or page addresses are placed, I put # for now.

You have also noticed that inside the <nav> tag we have a class name nav-bar. written in this format <nav class="nav-bar">. The class name is what we are going to be using as a target when styling it with CSS, a well built website must have many sections with different designs, therefore, a class name or Id name is what tells CSS exactly what to design. Feel free to use any name!

   
<body>
  <nav class="nav-bar">
    <h1>LOGO</h1>
      <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About us</a></li>
        <li><a href="#">Services</a></li>
        <li><a href="#">Portfolio</a></li>
      </ul>
  </nav> 
</body>

if you run this codes on your browser, this is what it will look like, we have structured the with HTML but it's not normal because we've not applied any design yet. Now we are going to design it by targeting our class name nav-bar in CSS.


CSS normally uses a HTML class or Id name to design a particular section of the website. It uses a dot '•' in front of the class name like this: .nav-bar and uses '#' for Id name like this: #nav-bar, then followed by open and close curly braces { } in which you will write your codes.

     
/* In this code, we targeted our class name 'nav-bar' with a dot infront of it,
  but we did not put anything infront of the 'body' tag because
  it is neither a class nor Id, it is just a direct tag. */
      
       
body {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
    
.nav-bar {
  width: 100%;
  margin: 0;
}
    
.nav-bar > h1 {
  position: absolute;
  font-family: Arial, Helvetica, sans-serif;
  color: white;
  padding-left: 80px;
}
    
.nav-bar > ul {
  display: flex;
  margin: 0;
  width: 100%;
  height: 70px;
  justify-content: right;
  background-color: rgb(39, 112, 112);
}
    
.nav-bar > ul > li {
  list-style: none;
  line-height: 60px;
  font-size: 20px;
  padding: 0 10px;
  position: relative;
 right: 100px;
}

This is how our now looks after we have applied some CSS styles. The body is the body of the web page, in which everything on the page is placed. We used 'margin 0' because we want our web page to have full-width, while 'padding 0' tells it not to adjust any element within the page, unless we apply that directly from the element itself. The 'box-sizing' defines how the width and height of an element are calculated, therefore, the value 'border-box' adds extra border and padding to the width and height of any box element that could be created.

Let us now change the hyperlinks color and remove the underlines. Attention please! whenever you see something like this .nav-bar > ul > li > a { }, it means that your main target here is the <a> tag, we are trying to tell CSS that the 'a' is inside the <li> tag, while the 'li' is inside the <ul> tag, and the 'ul' is the child of our main target .nav-bar. The reason is because there may be many <> tags in different pages or sections of your website, if you don't tell CSS the exact target, it will style all the 'a' on your site, and you should know, this hyperlink 'a' also appears in the footer section of every website, which we will be creating next. Let's move on! 

    
/* We simply changed the link coror, font size, and apply a hover effect.*/

   
.nav-bar > ul > li > a {
  text-decoration: none;
  color: white;
  font-family: Arial, Helvetica, sans-serif;
  text-transform: uppercase;
  font-size: 18px;
}
   
.nav-bar > ul > li > a:hover {
  color: yellow;
  transition: 0.4s;
}

A :hover effect and the rest of them are known as CSS pseudo classes, they are keywords added after a HTML class/Id name, or tags to specify a special state of that very selector. Remember, tags, classes or Id names are also called 'selectors' in CSS, so, don't think of that as something else. We added a 'hover' to our <a> tag/selector, and we applied yellow color to it so that when someone hovers their mouse on the links, the action will take effect. The 'transition' is set 0.4seconds, it makes the hover effect not to act abruptly, but in a cool manner. Don't worry, you will learn how these things works as we move on to the next step.

We have successfully created our , if you are reading this now it means that you have been following this tutorial step by step, so, I must congratulate you for haven learned how to build a navigation bar using HTML and CSS.

After creating the navigation bar for your website, the next thing that comes to mind is usually the footer. However, there is no rule about what to create first and last when building a website, but the reason why most developers create the navigation bar and the footer first may simply be beacause they are similar in regard to their contents.

A footer is the bottom section of a website, every website has it for it contains neccessary Information including a brief introduction to the site, quick links, and sitemap. It also contains a copyright notice, link to privacy policy, and so on. The purpose of a website footer is to help visitors by reminding them the navigation options that they may have not seen on top, due to the sub-menu that hides them from the main navigation bar, therefore, a footer plays an important role on the website for it enables developers to engage with their clients.

We are now going to build our website's footer that will have four columns.

  1. A column that will contain a section about the business or company, and also the brand logo.
  2. A column that will contain few links to more important pages and sections of the site.
  3. A column that will link the visitors outside of the site, for example; our social media Networks.
  4. A column that will contain policies and regulations of the site/company.

You could also see that we are having four lines of code inside of it wrapped with a <div> tag. div means "divide" in HTML, it is used for creating containers, columns, and several kinds of layout. The four divs we have here will create four columns, and we have given each of them a class name that describes its content. Remember, a class name is not a code, you can name it anything you want, but it is advised to always use a name that describes what its contents will be, the reason is because before a complete website is structured, you're going to write a lot of code, and it is obvious that you will always make changes to your website from time to time, even when published, therefore, it will be almost imposible to understand your code later if you don't use descriptive class and id names. please, take note on that!

Now, take a look at our code carefully, we have created contents inside each of the four divs. The first div is the introductory container that has three tag elements inside of it. The <h1> tag, which is where we write out title, About us. Followed by the <img> tag, which is where we place our Logo. This img tag has two attributes inside of it, the src=" " that helps us search images, videos, and other files inside our computer. The alt=" " attribute is where give alternative name to the file inside the <img> tag, which will only be visible on the website whenever the main image fails to load as a result of poor internet connectivity, it helps site visitors to know what was put there. Then thirdly, it has a p tag, this is used for writing a paragraph in HTML, just like the one you're reading now, I used the <p> tag to create it.

HTML attributes are special words used inside tags to place elements and modifies or control their behaviour. In the code examples you can see that the src attribute placed the logo, while the alt controls its behaviour, so, this is what attributes are. Don't worry, you will learn more about them as we move on.

The second div is our quick links, inside we have two tags, the h1 for the title, and the ul for the unodered lists. We explained the <ul> tag in our previous tutorial, the . But the <ul> itself has two tags inside of it, the <li> that lists the items, and the <a> that links the items to somewhere else. The a tag always has the attribute href=" " inside of it whose value will be the website, or page address where you want the clicker to land. We also talked about them in our previous tutorial.

Finaly, the third and the fourth div are similar to the second div. So, let us move on, but before that, always remember that every open tag must have a closing tag, for example; <p>My paragraph</p>. However, there are some HTML tags that do not require a closing tag, such tags are called void tags which normally have one or more attributes inside of them, you will learn more about them as we move on. And also, remember that every attribute must be in this format, html-reserved-words-for-attributes followed with equals mark =, and then, two double quotes " " where the value, or element will be placed. example; width="100%". The word width is an attribute, while 100% is the value. You can use any value according to your need.

Alright, we have structured our footer with HTML, we are Now going to design it with CSS, so that it will look appealing to the eyes.

We will be targeting the footer with the id name "footer-container", I used an id this time because i want to show you how to use it in CSS. But we use class names for the four inner containers, and we will be targeting them one after another. However, if you observe, you will notice that we are having two class names for one div, and that the extra class name inner-div is used for the four divs. The reason is simply because there is no need to keep repeating our codes in CSS, if there should be any style that will apply for the four divs, we don't have to rewrite our code in four places, we simply have to target the general class name, then, the style will apply on all, and if there should be a unique style, then, we target the unique class name.

So far so good, we're heading somewhere! You can now see how our footer looks after we have written few lines of codes in CSS.

Now you can understand what i was talking about, the footer was given an id name "footer-container", that is why we have to use # in front of it in CSS, instead of a "dot". Now, I want you to know something, width, height, and the rest of them are called attributes in HTML, but in CSS they are called properties, their values remain values, both in HTML, CSS, and even in JavaScript. So, don't be confused about that.

We give the width the value of 100% because we want it to be in full-width. We set the padding 0 30px. The 0 applies to the top and bottom, we don't want the four columns to have space in the top and bottom between them and their container the footer. The 30px applies to the left and right, we are giving 30px to the left and 30px to the right, that is the easiest way to use paddind in CSS, we could have written it this way (padding-top 0; padding-bottom 0; padding-left 30px; padding-right 30px;), it is still the same, but that is not the best way to write codes.

You can notice that the first structure was in a vertical alignment by default, but the second one is now horizontal, that happened because we set the display property "flex", and by default they will float to the left side, but the justify-content put them in the center.
